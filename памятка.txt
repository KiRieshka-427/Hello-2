ВНИМАНИЕ!!!!!!!!!!АЛГОРИТМ !!!!!!!!!!!!!!ВНИМАНИЕ

1) Заходим по общей ссылке
2) Нажимаем Fork - появится СВОЯ отдельная ветка
3) Далее открываем git-cmd.exe (должен быть установлен Git)
4) Переходим в рабочую папку на своём пк
5) Прописать git init
6) Связываем локальный и удалённый репозиторий(?) с помощью git remote add origin ссылка на свою ФОРКу
6) Задаём вводные
7) Создаём копию, если она ещё не создана - git clone https://...
8) Переносим файлы из удалённого места в локальное с помощью git pull origin main
9) Работаем с этими файлами по основному заданию
10.0) Опционально посмотреть несохранённые файлы через git status
11) Готовим все файлы к коммиту с помощью git add . (точка - ВСЕ файлы)
11) Коммит с помощью git commit -m "[№ задания] изменения/комментарии"
12) Заносим всё обратно в удалённый репозиторий с помощью git push origin main
13) На самом сайте делаем Contribute -> open pull request -> Create pull request -> Называем свою работу "[№ задания] изменения/комментарии" -> Create pull request
14) Наслаждаемся проделанной работой и молимся, что ничего не сломалось

(В дальнейшем пропускаем пункты 2, 5, 6, 7)

ГИТОВСКИЕ ДВИЖЕНИЯ

git init - показываем ГИТу, за чем следить

git remote add origin https://github.com/kit8nino/2024-MP.git - связывание локального и удалённого репозитория (по ссылке)

git clone https://github.com/kit8nino/2024-MP.git - создание копии удалённого в локальном

git status - показывает инфу обо всех НЕЗАКОММИЧЕННЫХ файлах в репозитории

git add . - подготовка к коммиту ВСЕХ файлов (точка)

git commit -m "бла-бла-бла" - коммитит готовые файлы (с комментарием)

git log - инфа обо всех коммитах

git checkout 460f2fe - возвращение к прошлой версии через хэш-код

git push origin master - загрузка изменений НА гитхаб (иногда вместо master ставится main)

git pull origin master - загрузка последних изменений С гитхаба (иногда вместо master ставится main)

git checkout - показывает все ветки
(звёздочкой * отмечается текущая ветка)

git branch feature - создание новой ветки feature (имя любое)

git branch -d feature - удаление ветки feature

git checkout feature - переход на ветку feature (имя любое)

git push origin feature - загрузка ветки на гитхаб

git merge feature - слияние ветки feature в master
(до этого сначала надо перейти в master)

git remote remove *название удалённого источника (origin)* - удаление источника

git remote -v - просмотр всех источников

git commit -m "Комментарий" --amend - добавление файла в коммит
